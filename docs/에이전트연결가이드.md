# Access - 에이전트 연결 가이드

모든 에이전트가 같은 할 일 리스트를 공유합니다. 작업이 완료되면 자동으로 체크됩니다.

## 연결 현황

| 에이전트 | 연결 방식 | 상태 | agent 이름 |
|---------|---------|------|-----------|
| **ZCode** | MCP (`access-todo`) | ✅ 등록 완료 | `zcode` |
| **OMP** | MCP (`access-todo`) | ✅ 등록 완료 | `omp` |
| **Hermes** | REST (curl skill) | ✅ skill 등록 완료 | `hermes` |
| **Claude Code** | MCP (동일) | ✅ 동일 방식 | `claude` |

## 전제 조건

`Access시작.bat` 실행 → API 서버 (`http://127.0.0.1:7878`) 켜져 있어야 함

---

## 자동 체크 시스템

**작업을 완료하면 에이전트가 자동으로 할 일을 체크합니다.**

```
에이전트가 작업 수행
  → "완료"/"끝"/"고쳤어" 인식
  → search_todos로 관련 할 일 검색
  → toggle_todo로 체크 ✓
  → 포스트잇 + 안드로이드에 실시간 반영
```

체크 조건:
- ✅ 사용자가 완료 의미 표현 + 실제 코드/파일 변경 완료
- ❌ 단순 질문, 작업 미완료, Access에 해당 할 일 없음

## 1. ZCode (MCP)

### 설정 위치
`C:\Users\USER-PC\.zcode\cli\config.json`

```json
"mcp": {
  "servers": {
    "access-todo": {
      "type": "stdio",
      "command": "C:\\Users\\USER-PC\\Desktop\\Access\\Access-MCP.exe",
      "enabled": true
    }
  }
}
```

### 자동 체크 규칙
`C:\Users\USER-PC\ZCodeProject\AGENTS.md`에 지시문 작성됨

### 사용 예시
```
(자연어 대화)
"K-STONKS 버그 수정해줘"
→ 에이전트가 코드 수정 후 toggle_todo 자동 호출
→ "✓ Access 포스트잇에서 체크했습니다"
```

## 3. Hermes Agent (REST API)

### 설정 위치
Skill: `C:\Users\USER-PC\AppData\Local\hermes\hermes-agent\skills\access-todo\SKILL.md`

Hermes는 터미널 명령어(curl)로 Access API를 호출합니다.

### 자동 체크 규칙
SKILL.md 안에 "작업 완료 시 access_check 호출" 지시문 포함

### 사용 예시
```
"hermes, 백엔드 시그널 룰 엔진 구현해줘"
→ Hermes가 코드 작성 후:
  1. access_search q="시그널 룰"
  2. access_check {id}
  3. "시그널 룰 엔진 작업을 체크했습니다 ✓"
```

### 수동 명령어 (Hermes 터미널에서)

```bash
# 할 일 조회
curl -s -H "X-Agent: hermes" http://127.0.0.1:7878/todos

# 추가
curl -s -X POST -H "X-Agent: hermes" -H "Content-Type: application/json" \
  -d '{"title":"새 작업","priority":"high","tags":["agent:hermes"]}' \
  http://127.0.0.1:7878/todos

# 체크 (id 필요)
curl -s -X POST -H "X-Agent: hermes" http://127.0.0.1:7878/todos/{id}/toggle
```

## 카테고리 (tags)

할 일의 `tags`에 에이전트 식별자를 넣으면 포스트잇에서 자동 그룹화:

| tags | 포스트잇 그룹 |
|------|-------------|
| `["agent:common"]` | 📌 공통 |
| `["agent:hermes"]` | 🤖 hermes |
| `["agent:zcode"]` | ⚡ zcode |
| `["agent:omp"]` | 🛠️ omp |
| `["agent:user"]` | 👤 user |

## 데이터 흐름

```
ZCode (MCP) ─────┐
OMP   (MCP) ─────┤
Hermes (REST) ───┤→ API 서버 (7878) → GitHub Gist → 포스트잇 + 안드로이드
사용자 (위젯) ───┘
```

모든 변경이:
- 포스트잇 위젯에 15초 내 표시
- GitHub Gist에 자동 동기화
- `created_by` / `completed_by`로 추적

## 문제 해결

| 증상 | 해결 |
|------|------|
| MCP 도구 안 보임 | ZCode/OMP 재시작 |
| Hermes가 Access 인식 못 함 | API 서버 실행 확인 (`curl http://127.0.0.1:7878/health`) |
| 포스트잇에 반영 안 됨 | `POST http://127.0.0.1:7878/sync` 로 수동 동기화 |
| 자동 체크 안 됨 | AGENTS.md / SKILL.md / skills/*.md 규칙 파일 확인 |
