# Access 사용자 가이드

> 에이전트와 나를 연결하는 포스트잇 TODO 시스템
> 이 가이드는 사용자가 Access를 처음부터 끝까지 사용하는 방법을 단계별로 설명합니다.

---

## 📖 Access가 뭔가요?

Access는 바탕화면에 뜨는 포스트잇입니다. **나와 에이전트(ZCode, Hermes, OMP)가 같은 할 일 리스트를 공유**합니다.

- 내가 포스트잇에 할 일을 적으면 → 에이전트가 봅니다
- 에이전트가 할 일을 추가하면 → 내 포스트잇에 바로 나타납니다
- 에이전트가 작업을 끝내면 → 체크 + "무엇을 했는지" 요약이 표시됩니다

**핵심**: 에이전트가 자유롭게 일하지만, Access가 "뭘 했고, 뭘 해야 하는지" 정리해줍니다.

---

## 🚀 시작하기

### 1. Access 켜기

바탕화면의 **`Access시작.bat`** 을 더블클릭하세요.

```
[1/3] API 서버 시작 중...
[2/3] 포스트잇 4개 창 시작 중...
[3/3] 에이전트 감시 시작...
완료! 4개 포스트잇이 바탕화면에 나타납니다.
```

화면에 4개 포스트잇이 2×2로 나타납니다:

```
┌──────────┐ ┌──────────┐
│🤖 Hermes │ │🛠️ OMP    │
│ (핑크)   │ │ (블루)   │
└──────────┘ └──────────┘
┌──────────┐ ┌──────────┐
│⚡ ZCode  │ │👤 내 할일 │
│ (민트)   │ │ (노랑)   │
└──────────┘ └──────────┘
```

### 2. 포스트잇 조작법

| 조작 | 방법 |
|------|------|
| **이동** | 제목 영역(🤖 Hermes 등)을 마우스로 누르고 드래그 |
| **항상 위 켜기/끄기** | 📌 버튼 클릭 (켜져있으면 다른 창 위에 항상 표시) |
| **접기/펼치기** | ▴ 버튼 클릭 |
| **최소화** | ⚊ 버튼 클릭 |
| **닫기** | ✕ 버튼 클릭 |
| **카테고리 접기** | 카테고리명(📂 ...) 한 번 클릭 |
| **카테고리 이름 변경** | 카테고리명 더블클릭 또는 ✏️ 버튼 |

## 📝 할 일 작성 규칙

### 할 일 추가하기

하단의 **"+ 할 일"** 버튼을 클릭:

```
┌─────────────────────┐
│ 할 일 제목           │
│ 메모 (선택)          │
│ 카테고리: [선택]     │
│ 우선순위: [선택]     │
│ [추가]  [취소]       │
└─────────────────────┘
```

**작성 요령:**

| 항목 | 설명 | 예시 |
|------|------|------|
| **제목** | 핵심만 간결하게 | `장보기`, `로그인 버그 수정` |
| **메모** | 상세 내용, 링크, 파일 위치 | `우유, 계란, 식빵` |
| **카테고리** | 프로젝트/영역별 분류 | `버그 수정`, `기능 개발` |
| **우선순위** | 🔴높음 / 🟡보통 / 🟢낮음 | 긴급 버그 = 🔴 |

### 좋은 할 일 vs 나쁜 할 일

| ✅ 좋은 예 | ❌ 나쁜 예 |
|-----------|----------|
| `K-STONKS: job_select.lua nil 버그 수정` | `버그 수정` (어떤 버그?) |
| `DNF 가이드: 42개 직업 페이지 SEO 점검` | `작업하기` (무슨 작업?) |
| `백엔드: 시그널 룰 엔진 POST /rules 구현` | `코드` (너무 짧음) |

### 카테고리 만들기

하단의 **"📂 카테고리"** 버튼 클릭 → 이름 입력:

```
추천 카테고리 구조:
📂 버그 수정    ← 긴급한 버그
📂 기능 개발    ← 새 기능
📂 문서         ← 문서/가이드
📂 리팩터링     ← 코드 정리
📂 기타         ← 잡일
```

카테고리 관리:
- **✏️** 이름 변경
- **▲ ▼** 순서 변경
- **🗑️** 삭제 (할 일은 미분류로 이동)

## 🤖 에이전트에게 일 시키기

### 에이전트가 자동으로 하는 일

에이전트(ZCode, Hermes, OMP)는 Access에 연결되면:

1. **세션 시작 시** → 현재 할 일을 점검 (`access_review`)
2. **작업 완료 시** → 체크 + 요약 기록 (`complete_todo`)
3. **새 작업 발견 시** → 자동으로 할 일 추가 (`add_todo`)

### 에이전트에게 지시하는 방법

**ZCode에게 (이 세션에서):**
```
"K-STONKS-V2 버그 수정해줘"
→ ZCode가 코드 수정 후 complete_todo로 "무엇을 했는지" 기록

"DNF 가이드 새 페이지 추가해줘"
→ ZCode가 작업 후 요약과 함께 체크
```

**Hermes에게:**
```
"hermes, 백엔드 시그널 룰 엔진 구현해줘"
→ Hermes가 코드 작성 후 access_complete로 요약 기록
```

### 에이전트가 남긴 작업 요약 확인

완료된 할 일의 메모를 보면 됩니다:
```
☑ 🔴 K-STONKS: job_select.lua nil 버그 수정
   ✅ ✓ main.lua 1867줄 H 변수 nil 체크 추가, job_select.lua 51줄 jc 초기화
   체크: zcode
```

### 내 할 일을 에이전트에게 넘기기

👤 내 할 일 포스트잇에 추가하고, 에이전트에게 말합니다:
```
"K-STONKS 버그 수정해줘. Access에 할 일 올려놨어."
→ 에이전트가 search_todos로 찾아서 작업
```

또는 에이전트에게 직접 지시:
```
"ZCode, job_select.lua 버그 Access에 추가하고 수정해줘"
```

## 📊 진행상황 확인하기

에이전트에게 물어보세요:
```
"지금 진행상황 어때?"
→ 에이전트가 access_review 호출하여 보고
```

반환되는 정보:
```
진행률: 11/19 (57%)
완료: 11개 (요약 포함)
남음: 8개 (high 2, medium 2, low 4)
긴급 다음: job_select.lua nil, main.lua H 크래시
```

## 📂 프로젝트 온보딩 (새 프로젝트 연동)

새 프로젝트를 에이전트와 시작할 때:

### 방법 1: .access/rules.json (권장)

프로젝트 폴더에 `.access/rules.json`을 만듭니다:

```json
{
  "project": { "id": "my-project", "name": "내 프로젝트" },
  "categories": ["버그 수정", "기능 개발"],
  "starter_todos": [
    { "title": "README 작성", "priority": "medium", "category": "기능 개발" },
    { "title": "인증 버그 수정", "priority": "high", "category": "버그 수정" }
  ]
}
```

에이전트에게: `"이 프로젝트 Access에 온보딩해줘"`

### 방법 2: 에이전트에게 직접 요청
```
"이 프로젝트 구조 분석해서 Access에 카테고리랑 할 일 만들어줘"
```

## 🔄 동기화

- 모든 할 일은 **GitHub Gist**에 자동 저장됩니다
- 포스트잇은 **15초마다** 자동 새로고침
- 에이전트가 추가한 항목 → **황금색 깜빡임** + 헤더에 🔴 배지
- 안드로이드 앱에서도 같은 리스트 공유 (APK 다운로드: GitHub Releases)

## 🛑 종료하기

### 포스트잇만 닫기
각 포스트잇의 **✕ 버튼** 클릭

### Access 전체 종료
에이전트를 모두 종료하면 Access도 자동으로 종료됩니다.
또는 작업 관리자에서 `Access-API.exe` 종료.

### 다시 시작
`Access시작.bat` 더블클릭.

## ❓ 자주 묻는 질문

**Q: 에이전트가 추가한 할 일이 안 보여요**
→ 15초를 기다리거나, 에이전트에게 "access 동기화해줘"라고 하세요.

**Q: 포스트잇 위치가 저장이 안 돼요**
→ 현재 버전에서는 재시작 시 기본 위치로 돌아갑니다. (추후 개선 예정)

**Q: 다른 PC에서도 쓸 수 있나요?**
→ 같은 GitHub Token + Gist ID를 입력하면 됩니다. 안드로이드도 마찬가지.

**Q: 에이전트가 작업을 안 끝냈는데 체크했어요**
→ 포스트잇에서 체크를 해제하면 됩니다. (체크박스 클릭)

**Q: 카테고리가 너무 많아져요**
→ 🗑️ 버튼으로 삭제하세요. 할 일은 미분류로 자동 이동합니다.
