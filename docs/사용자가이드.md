# Access 완전 가이드 — 규칙과 사용법

> **Access = "에이전트 정리 가이드 시스템"**
> 에이전트가 자유롭게 일하지만, Access가 규칙에 따라 정리해서
> **"지금 뭘 해야 하는지"를 사용자에게 알려줍니다.**

---

## 📖 Access가 하는 일 (30초 설명)

에이전트(ZCode, Hermes, OMP)가 코드를 고치고, 파일을 만들고, 테스트를 합니다.
그 과정에서 **뭘 했는지, 뭘 더 해야 하는지**를 Access에 기록합니다.

Access는 이 기록을 모아서:
1. **진행률**을 보여주고 (40% 완료!)
2. **지금 할 일**을 추천하고 (👉 이거 하세요)
3. **내가 해야 할 일**을 따로 표시합니다 (👤 노란 포스트잇)

---

## 🔴 Access의 5가지 정리 규칙

### 규칙 1: 우선순위로 "지금 뭘 할지" 결정

모든 할 일에는 중요도가 있습니다:

| 표시 | 의미 | 언제? | 예시 |
|------|------|-------|------|
| 🔴 빨간 점 | **지금 당장** | 크래시, 서비스 중단, 데이터 손실 | 게임이 켜지지 않음 |
| 🟡 노란 점 | **오늘 중** | 기능 개발, 일반 버그, 개선 | 버튼이 안 눌림 |
| 🟢 초론 점 | **시간 날 때** | 문서, 정리, 선택적 개선 | README 업데이트 |

**포스트잇 상단의 "👉 지금 이거 하세요"** 에 빨간 점부터 보여줍니다.

### 규칙 2: 폴더(카테고리)로 정리

할 일을 프로젝트나 작업 종류별로 묶습니다:

```
📂 버그 수정     ← 고쳐야 하는 것들
📂 기능 개발     ← 새로 만드는 것들
📂 문서/정리     ← 문서 작업
📂 기타          ← 분류 안 된 것들
```

같은 종류끼리 묶여 있어서 **한 프로젝트에 뭘 해야 하는지** 한눈에 봅니다.

### 규칙 3: 에이전트별로 분리

에이전트마다 자기 포스트잇이 따로 있습니다:

```
🤖 Hermes (핑크) → Hermes가 하는 일
🛠️ OMP (파랑)   → OMP가 하는 일
⚡ ZCode (초록)  → ZCode가 하는 일
👤 내 할 일 (노랑) → 내가 직접 해야 할 일
```

**노란 포스트잇만 보면 "내가 뭘 해야 하는지" 알 수 있습니다.**

### 규칙 4: 완료하면 요약을 남김

에이전트가 작업을 끝내면:

```
☑ 🔴 K-STONKS 버그 수정
   ✅ main.lua 1867줄 nil 체크 추가, job_select.lua 수정
   체크: zcode
```

→ **"뭘 했는지"**가 요약으로 남아서, 다음에 봐도 무엇을 했는지 압니다.

### 규칙 5: 에이전트 → 사용자에게 일 위임

에이전트가 "이건 내가 못 해, 사용자가 해야 해"라고 판단하면:

```
👤 내 할 일에 자동 추가:
  🔴 결제: AWS 크레딧 충전 ($50)
  🟡 승인: PR #42 머지 여부 확인
```

→ 에이전트가 나에게 **"이거 해주세요"**라고 요청하는 겁니다.

---

## 👀 포스트잇 읽는 법 (비전공자용)

### 1단계: 진행률 바 확인
```
████████░░░░░░░░ 8/20 (40%)
```
→ "전체 20개 중 8개 끝났구나. 아직 12개 남았네"

### 2단계: "지금 이거 하세요" 확인
```
👉 지금 이거 하세요
  🔴 긴급 K-STONKS 버그 수정
  🟡 보통 DNF 가이드 검수
```
→ "이 두 개부터 하면 되겠다"

### 3단계: 폴더별로 확인
```
📂 버그 수정 (3)
  🔴 main.lua 크래시
  🟡 job_select nil
  🟢 주석 정리
```
→ 빨간 점부터 처리하면 됩니다

### 4단계: 체크하면 완료!
할 일을 끝내면 체크박스 클릭 → 진행률 바가 올라갑니다

---

## ✍️ 할 일 추가하는 법

"+ 할 일" 버튼을 누르면:

```
무엇을 해야 하나요? [K-STONKS 차트 버그 수정     ]
메모 (선택)        [대형 차트가 안 그려짐        ]
폴더: [버그 수정 ▾]
중요도:
  🔴 긴급 (지금 해야 함)     ← 당장 해야 하는 거
  🟡 보통 (오늘 중)          ← 오늘 안에
  🟢 여유 (시간 날 때)       ← 천천히
[추가]
```

### 제목 쓰는 규칙
| ✅ 좋은 예 | ❌ 나쁜 예 | 이유 |
|-----------|----------|------|
| "K-STONKS 차트 버그 수정" | "버그" | 뭘 고쳐야 하는지 모름 |
| "DNF 가이드 SEO 점검" | "작업" | 무슨 작업인지 모름 |
| "장보기 - 우유, 계란" | "장보기" | 뭘 사야 하는지 모름 |

**규칙: 제목만 봐도 "아, 이거구나" 하고 알 수 있게 쓰기**

## 🤖 에이전트에게 일 시키는 법

### 그냥 말로 시키면 됩니다

```
"K-STONKS 버그 수정해줘"
→ ZCode가 수정 → 완료하면 "뭘 했는지" 요약 표시

"백엔드 시그널 룰 엔진 구현해줘"
→ Hermes가 구현 → 완료 요약 표시
```

### 에이전트가 완료하면?
```
☑ 🔴 시그널 룰 엔진 구현
   ✅ POST /rules 엔드포인트, 3개 테스트, auth 권한 검증 추가
   체크: hermes
```

### 에이전트가 나에게 일을 주면?
노란 포스트잇(👤 내 할 일)에 자동 추가:
```
👤 내 할 일:
  🔴 AWS 크레딧 충전 ($50)    ← 에이전트가 "이거 해주세요"라고
  🟡 PR #42 머지 승인          ← 사용자 결정이 필요한 것들
```

## 💡 더 쉽게 쓰는 제안

### 제안 1: 매일 아침 "오늘의 추천" 확인

포스트잇의 "👉 지금 이거 하세요"를 매일 아침 먼저 보세요.
에이전트가 정리해둔 우선순위라서, **뭘 먼저 해야 할지 고민할 필요가 없습니다.**

### 제안 2: 폴더를 프로젝트 이름으로

```
📂 K-STONKS-V2     ← 게임 프로젝트
📂 DNF 가이드      ← 웹사이트 프로젝트
📂 OneClick        ← 기획 프로젝트
```

프로젝트별로 폴더를 만들면, **어떤 프로젝트에 뭘 해야 하는지** 한눈에 알 수 있습니다.

### 제안 3: 노란 포스트잇을 "알람"으로 쓰기

에이전트가 노란 포스트잇에 추가한 항목은 **"에이전트가 나에게 요청한 것"**입니다.
빨간 점(🔴)이 뜨면 빨리 처리하세요.

### 제안 4: 완료 요약을 "업무 일지"로 활용

완료된 항목의 ✅ 요약을 보면, **그날 무슨 작업을 했는지** 한눈에 알 수 있습니다.
나중에 보고서나 회고를 쓸 때 유용합니다.

### 제안 5: "진행상황 어때?"라고 물어보기

에이전트에게 물어보면 전체 요약을 보여줍니다:
```
"진행상황 어때?"
→ 에이전트가 access_review로 정리해서 보고
```

## ❓ FAQ

**Q: 완료된 항목이 안 보여요**
→ "🙈 완료 항목 숨기기"를 클릭하면 보입니다.

**Q: 에이전트가 추가한 게 5초가 지나도 안 떠요**
→ API 서버가 꺼져 있을 수 있습니다. Access를 다시 시작하세요.

**Q: "지금 이거 하세요"에 아무것도 안 떠요**
→ 빨간 점(🔴) 할 일이 없다는 뜻입니다. 긴급한 일이 없으니 여유롭게 하셔도 됩니다!

**Q: 에이전트가 내 할 일에 뭔가를 추가했어요**
→ 에이전트가 "이건 사용자가 해주세요"라고 요청한 것입니다. 노란 포스트잇을 확인하세요.

**Q: 폴더가 너무 많아요**
→ 🗑️로 삭제하세요. 할 일은 📦 미분류로 자동 이동합니다.

**Q: 진행률이 안 올라가요**
→ 할 일의 체크박스를 클릭해야 완료로 처리됩니다.
