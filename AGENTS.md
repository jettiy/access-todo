# Access Todo — 에이전트 지시 규칙

이 프로젝트는 Access 포스트잇 TODO 시스템과 연동되어 있습니다.
`access-todo` MCP 서버가 ZCode에 등록되어 있습니다.

## 🔴 필수 규칙: 작업 완료 시 요약과 함께 체크

사용자가 요청한 작업을 완료한 후 (코드 수정, 버그 수정, 문서 작성 등):

### 1단계: 무엇을 변경했는지 분석
```
- git diff 또는 변경된 파일 목록 확인
- 핵심 변경사항을 1-2문장으로 요약
```

### 2단계: complete_todo 호출 (toggle_todo가 아님!)
```
complete_todo(id="<할일id>", summary="<작업 요약>", agent="zcode")
```

**summary 작성 요령:**
- ✅ 좋은 예: `job_select.lua 51줄 nil 체크 추가, main.lua 1867줄 H 변수 초기화`
- ✅ 좋은 예: `백엔드 시그널 룰 엔진 POST /rules 구현, 테스트 3개 추가`
- ❌ 나쁜 예: `작업 완료` (너무 모호)
- ❌ 나쁜 예: `코드 수정함` (무엇을 했는지 불명홭)

### 3단계: 사용자에게 알림
```
"✓ '<할일제목>' 완료: <요약>"
```

## 체크 조건
- ✅ 사용자가 "완료", "끝", "done", "고쳤어" 등 완료 의미 표현 + 실제 변경 완료
- ❌ 단순 질문/조회
- ❌ 작업 미완료
- Access에 해당 할 일이 없음 → 새 추가 여부 확인

## 새 작업 발견 시
버그 발견, 새 기능 필요 등:
- `add_todo` (agent: "zcode", tags: ["agent:zcode"]) 로 자동 추가
- 우선순위는 상황에 맞게 판단 (긴급 버그 = high)
