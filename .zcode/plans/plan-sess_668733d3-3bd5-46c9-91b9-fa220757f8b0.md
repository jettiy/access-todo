## 목표
`C:\Users\USER-PC\Desktop\Trading`에 Vibe-Trading 소스를 git clone하고, **독립 venv**로 개발 환경을 구축합니다. 기존 pip 패키지(`vibe-trading-ai 0.1.10`, base 환경)는 그대로 보존하여 참고용으로 남겨둡니다. 두 환경이 충돌하지 않습니다.

## 환경 분리 구조
- **base 환경** (miniconda3): 기존 pip 패키지 유지 → `vibe-trading` 치면 구버전 실행 (참고용)
- **Desktop\Trading\.venv** (신규): git clone + `pip install -e .` → 소스 수정 즉시 반영되는 개발 버전
- Python 패키지/의존성은 venv로 완전 분리. 기존 환경에 영향 없음.
- 공유 데이터(`~/.vibe-trading/`의 `.env`, `sessions.db`, `memory/`)는 그대로 유지 — 기존 OpenRouter 설정 재사용

## 실행 단계

### 1단계: git clone
```
cd C:\Users\USER-PC\Desktop\Trading
git clone https://github.com/HKUDS/Vibe-Trading.git .
```
(`.` 붙여서 폴더 자체에 바로 받음. 현재 빈 폴더라 가능)

### 2단계: 독립 venv 생성
```
python -m venv .venv
```
Python 3.11 사용 (요구사항 충족)

### 3단계: 패키지 개발 모드 설치
```
.venv/Scripts/python -m pip install -e .
```
`-e`(editable) 모드 → 소스 수정 시 reinstall 없이 즉시 반영

### 4단계: 환경 설정 (.env)
- `agent/.env.example` → `agent/.env` 복사
- 기존 `~/.vibe-trading/.env`의 OpenRouter 설정(provider, API key, model, base URL)을 `agent/.env`로 복사
- README의 `.env` 검색 순서(`~/.vibe-trading/.env` → `agent/.env`)상 글로벌 설정도 읽히지만, 명시적으로 로컬에도 두어 개발 환경 독립성 강화

### 5단계: 프론트엔드 의존성 설치
```
cd frontend && npm install && cd ..
```
React 19 + Vite + TypeScript

### 6단계: CLI 동작 확인
```
.venv/Scripts/vibe-trading --version   # 버전 확인
.venv/Scripts/vibe-trading --help      # 명령 확인
```
→ 개발 버전이 정상 설치됐는지 검증

### 7단계: 백엔드 서버 실행 (포트 8899)
```
.venv/Scripts/vibe-trading serve --port 8899
```
FastAPI 서버. `/docs`에서 API 확인 가능.

### 8단계: 프론트엔드 개발 서버 실행 (포트 5899)
```
cd frontend && npm run dev
```
별도 터미널. Vite 개발 서버가 8899 백엔드로 API 프록시.

### 9단계: 웹 UI 접속 확인
- 브라우저에서 `http://localhost:5899` 접속
- 채팅 UI, 세션, 기능 동작 확인

## 주의사항
- `pip install -e .` 중 WeasyPrint 등 일부 시스템 의존성에서 경고가 날 수 있음 (PDF 기능). 핵심 기능엔 영향 없음.
- OpenRouter API 키는 기존 설정에서 안전하게 복사 (코드/로그에 노출 금지)
- base 환경의 기존 패키지는 절대 건드리지 않음
